# PRD — KMG Eurobond Investor Presentation Site

## 1. Обзор проекта

**Название:** KMG Eurobond Investor Presentation — Web Showcase
**Тип:** Статический веб-сайт (GitHub Pages совместимый)
**Назначение:** Презентационный лендинг и точка доступа к инвесторской презентации JSC NC KazMunayGas для готовящегося выпуска евробондов.

**Проблема, которую решает проект:**
Инвестиционная команда КМГ нуждается в быстром, презентабельном способе делиться инвесторской презентацией с потенциальными покупателями евробондов, аналитиками и партнёрами по размещению — без необходимости пересылать файлы по почте, использовать защищённые дата-румы для предварительного ознакомления, или объяснять структуру документа каждый раз заново.

**Решение:**
Одностраничный статический сайт, который:
- визуально представляет содержание презентации (обзор по слайдам/секциям);
- даёт возможность скачать актуальный .pptx одним кликом;
- работает полностью на статике (HTML/CSS/JS), без бэкенда — размещается на GitHub Pages бесплатно.

| Цель | Метрика |
|---|---|
| Быстрый доступ к презентации | Скачивание .pptx в ≤ 2 кликов с главной страницы |

## 3. Целевая ауд. & пользовательские сценарии

**Аудитория:**
- Институциональные инвесторы и аналитики, рассматривающие участие в размещении евробондов КМГ
- Организаторы размещения (букраннеры), сверяющие материалы
- Внутренняя IR-команда КМГ, использующая ссылку в переписке с инвесторами

**User stories:**
1. *Как инвестор*, я хочу увидеть ключевые цифры (EBITDA, рейтинги, долг) до того, как скачаю презентацию, чтобы решить, стоит ли тратить время на полный документ.
2. *Как аналитик*, я хочу пролистать превью всех слайдов в браузере, не скачивая файл, чтобы быстро сориентироваться в структуре.
3. *Как IR-менеджер КМГ*, я хочу разослать одну ссылку, которая всегда ведёт на актуальную версию презентации.
4. *Как любой пользователь*, я хочу скачать .pptx одним кликом, чтобы открыть его в PowerPoint и работать дальше (комментировать, редактировать).

## 4. Функциональные требования

### 4.1 Главная страница (Hero)
- Название эмитента (JSC NC KazMunayGas), тип документа (Eurobond Investor Presentation)
- Ключевые показатели прямо в hero-секции: Revenue, EBITDA, кредитные рейтинги (как тизер)
- Основная CTA-кнопка: **«Скачать презентацию (.pptx)»**
- Вторичная CTA: **«Смотреть слайды»** (скролл к галерее)

### 4.2 Блок ключевых показателей (Key Metrics Strip)
- 4–6 карточек с главными цифрами FY2025 (Revenue, EBITDA, Net Debt, Net Debt/EBITDA, Credit Ratings)
- Данные читаются из единого JSON-файла конфигурации (`data.json`), чтобы их можно было обновлять без правки HTML

### 4.3 Содержание презентации (Table of Contents)
- Список из 21 слайда / 13 разделов с названиями секций, сгруппированный по логическим блокам (как Agenda-слайд в самой презентации)

### 4.4 Галерея слайдов (Slide Viewer)
- Превью каждого слайда (изображение, сгенерированное из .pptx)
- Лайтбокс / увеличение по клику
- Возможность пролистывать стрелками (prev/next)

### 4.5 Скачивание файла
- Кнопка скачивания указывает на актуальный файл `KMG_Eurobond_Investor_Presentation.pptx`, лежащий в репозитории
- Дополнительно: кнопка «Скачать PDF» (опционально, если PDF-версия также публикуется)

### 4.6 Футер
- Disclaimer (как на слайде 2 презентации) — сокращённая версия с упоминанием, что это материал для обсуждения
- Контакты IR (ir@kmg.kz, телефон, ссылка на kmg.kz/investors)
- Дата последнего обновления презентации

## 5. Нефункциональные требования

- **Хостинг:** GitHub Pages (статические файлы, без сборки/без Node на сервере)
- **Адаптивность:** Mobile-first, корректное отображение на экранах от 360px
- **Производительность:** Изображения слайдов — компрессированные JPEG, lazy-loading
- **Доступность:** alt-тексты для всех изображений слайдов, контрастность согласно WCAG AA
- **Без бэкенда:** никаких серверных вызовов, баз данных, API ключей в клиентском коде
- **Совместимость с браузерами:** последние версии Chrome, Safari, Firefox, Edge

## 9. Визуальный стиль (Design System)

Сайт должен зеркалить фирменный стиль самой презентации:

| Токен | Значение |
|---|---|
| `--petrol` | `#0E4D44` (основной, тёмно-зелёный) |
| `--petrol-dark` | `#0A3A33` |
| `--petrol-light` | `#16695C` |
| `--gold` | `#C9A24B` (акцент) |
| `--gold-light` | `#E4CD8B` |
| `--paper` | `#F7F6F2` (светлый фон контента) |
| `--ink` | `#1B2421` (основной текст) |
| `--ink-soft` | `#4B5A56` (вторичный текст) |
| Заголовочный шрифт | Serif (Cambria / Georgia в вебе) |
| Основной шрифт | Sans (Calibri / system-ui в вебе) |

Принципы:
- Тёмные секции (hero, summary, ratings) на petrol-фоне с золотым акцентом — как в самой презентации
- Светлые секции (контент, таблицы) на paper-фоне
- Без декоративных цветных полос/страйпов по краям карточек (как и в самой презентации)
- Карточки — мягкие тени, скругления, не плоский material-design

## 10. Процесс обновления контента

1. Изменения в данных → правка соответствующего `slides_part*.js` в `/build`
2. Пересборка: `node build/build.js` → новый `.pptx`
3. Рекомпрессия: `python scripts/rezip.py`
4. Регенерация превью: LibreOffice → PDF → `pdftoppm` → `assets/slides/slide-NN.jpg`
5. Обновление `data.json`, если изменились ключевые метрики
6. Commit + push в `main` → GitHub Pages автоматически передеплоит сайт

## 11. Ограничения и риски

- Презентация — шаблон ("Strictly Private & Confidential — Template for Discussion Purposes"), финальные условия выпуска (объём, купон, тенор) пока не определены — сайт должен явно это отражать в disclaimer-блоке.
- Сайт не должен публично индексироваться поисковиками до решения IR-команды о публичном раскрытии (рекомендуется `robots.txt` с disallow или `noindex` meta, если доступ должен быть ограничен по ссылке).
- Все данные привязаны к моменту публикации (FY2025 пресс-релиз); при выходе новых квартальных результатов контент требует обновления.
